| 필드 | 타입 | 필수 | 설명 |
|---|---|---|---|
| `type` | TEXT | ✅ | `INCOME` / `EXPENSE` |
| `amount` | INTEGER | ✅ | 금액 (원 단위) |
| `categoryId` | FK | ✅ | 카테고리 참조 |
| `date` | TEXT (ISO) | ✅ | 거래 발생일 |
| `memo` | TEXT | - | 거래 메모 (최대 200자) |
| `paymentMethod` | TEXT | - | `CASH` / `CARD` / `TRANSFER` / `OTHER` |

## ⏰ F03. 반복 거래 자동화 (고정 수익·지출)

| ID | 기능 | 우선순위 | 설명 | 상태 |
|---|---|---|---|---|
| F03-1 | 반복 거래 등록 | P0 | 자동 등록할 고정 거래 설정 (DAILY/WEEKLY/MONTHLY/YEARLY) | ✅ |
| F03-2 | 반복 거래 목록 | P0 | 활성/비활성 반복 거래 관리, 드래그 정렬 | ✅ |
| F03-3 | 자동 실행 | P0 | WorkManager로 미처리 반복 거래 일괄 등록 | ✅ |
| F03-4 | 반복 거래 수정 | P1 | TransactionFormFragment에서 통합 수정 (recurringId NavArg) | ✅ |
| F03-5 | 반복 거래 비활성화 | P1 | 일시 중단 (삭제하지 않고 비활성) | ✅ |
| F03-6 | 실행 이력 확인 | P1 | 자동 등록된 거래에 `is_auto` 표시 | ✅ |

### 반복 거래 예시

| 이름 | 유형 | 금액 | 실행일 | 카테고리 |
|---|---|---|---|---|
| 월급 | 수입 | 3,500,000원 | 매월 1일 | 급여 |
| 넷플릭스 | 지출 | 17,000원 | 매월 15일 | 문화/여가 |
| 월세 | 지출 | 500,000원 | 매월 25일 | 주거/통신 |
| 핸드폰 요금 | 지출 | 65,000원 | 매월 20일 | 주거/통신 |

> [!info] WorkManager 기반 자동 실행
> 앱 실행 시 즉시 실행 + **매일 1회 주기적 WorkManager** 등록.
> 앱을 오래 열지 않아도 기기가 켜져 있으면 백그라운드에서 자동 처리.
> 장기간 미실행 시 밀린 거래를 한꺼번에 등록 (빠진 달 모두 처리).

## 🤖 F08. AI 요약 (Gemini Nano)

| ID | 기능 | 우선순위 | 설명 |
|---|---|---|---|
| F08-1 | 월간 소비 요약 | P1 | 이번 달 소비 패턴을 자연어로 2~3문장 요약 |
| F08-2 | 절약 조언 | P1 | 지출 데이터 기반 절약 가능 항목 제안 |
| F08-3 | 카테고리 자동 추천 | P2 | 거래 메모 입력 시 카테고리 자동 제안 |
| F08-4 | 전월 대비 분석 요약 | P2 | 전월과 비교한 소비 변화를 자연어로 설명 |
| F08-5 | 기기 호환성 체크 | P0 | Gemini Nano 지원 여부 확인 → 미지원 시 기능 숨김 |

### AI 요약 예시

| 시나리오 | AI 출력 |
|---|---|
| **월간 요약** | "4월 총 지출 82만원 중 식비가 45%로 가장 큰 비중입니다. 전월 대비 카페 지출이 30% 증가했습니다." |
| **절약 조언** | "카페/간식 지출이 월 6만원으로 예산의 92%입니다. 커피를 주 3회로 줄이면 월 2만원 절약 가능합니다." |
| **카테고리 추천** | 메모 "스타벅스 아이스아메리카노" → 추천: `카페/간식` |

> [!tip] 온디바이스 = 프라이버시
> Gemini Nano는 기기 내에서 추론하므로 **재무 데이터가 외부 서버로 전송되지 않는다**.
> 인터넷 연결 없이도 AI 요약이 가능하다.

> [!warning] 지원 기기 제한
> Gemini Nano는 Pixel 8 Pro+, Samsung Galaxy S24+ 등 AICore 탑재 기기에서만 동작한다.
> 미지원 기기에서는 F08 전체가 비활성화되며, 기존 수치 기반 통계(F06)만 표시된다.
